# 中医理论融入问题分析与修正报告

## 执行摘要

**核心问题**：当前系统仅采集超声影像和基本信息（年龄、乳腺状况等），但数据库和 API 中设计了大量中医相关字段（体质、证型、舌象、脉象等），导致**没有辨证依据却显示中医诊断结果**，产生严重误导。

**建议**：立即移除或标记所有无依据的中医相关字段，避免临床误导。如确需中医功能，必须补充完整的四诊信息采集。

---

## 1. 现状深度分析

### 1.1 输入数据范围

**实际采集的数据**：
```
✅ 超声影像数据
   - BI-RADS 分级
   - 肿块特征（大小、形态、边界、回声、钙化、血流）
   
✅ 基本信息
   - 患者年龄
   - 性别
   - 既往乳腺病史
   - 家族史
   
❌ 缺失的中医四诊信息
   - ❌ 望诊：舌象（舌质、舌苔）、面色、形体
   - ❌ 闻诊：声音、气味
   - ❌ 问诊：详细症状、月经生育史、生活方式、饮食偏好
   - ❌ 切诊：脉象（弦、滑、涩、细等）
```

### 1.2 数据库设计问题

### 1.3 前端采集问题

#### 体质采集表单（无依据）

```tsx
// frontend/src/pages/Patient/Create.tsx
<Form.Item name="constitution" label="体质类型">
  <Select options={[
    { value: '气郁质', label: '🌸 气郁质 - 情绪抑郁...' },
    { value: '痰湿质', label: '💧 痰湿质 - 形体肥胖...' },
    // ... 其他体质
  ]}/>
</Form.Item>
```

**问题分析**：
1. **下拉选择** - 医生可随意选择，系统不验证
2. **无评估量表** - 没有中医体质分类与判定标准（中华中医药学会标准）
3. **无舌脉支持** - 体质判定需要舌象、脉象等信息，系统未采集
4. **非必填项** - 可以跳过，但一旦填写就作为"诊断依据"存储

#### 证型采集（无辨证过程）

```tsx
<Form.Item name="zheng_type" label="中医证型">
  <Select options={[
    { value: '肝郁气滞', label: '肝郁气滞证' },
    { value: '脾虚痰湿', label: '脾虚痰湿证' },
    // ... 其他证型
  ]}/>
</Form.Item>
```

**问题分析**：
1. **直接选择证型** - 没有辨证推理过程
2. **缺少症状采集** - 没有"胸闷"、"胁痛"、"情志抑郁"等症状清单
3. **缺少舌脉信息** - 没有舌质、舌苔、脉象等关键辨证依据
4. **无鉴别诊断** - 如何区分"肝郁气滞"和"肝郁化火"？

## 2. 中医辨证的正确流程

### 2.1 标准辨证流程

```
┌─────────────────────────────────────────────────┐
│              中医辨证全流程                       │
├─────────────────────────────────────────────────┤
│                                                 │
│  1️⃣ 望诊                                         │
│     ┌─ 舌诊：舌质（淡/红/绛/紫）                 │
│     │     舌苔（白/黄/腻/薄/厚）                 │
│     └─ 面色：（白/黄/红/青/黑）                  │
│                                                 │
│  2️⃣ 闻诊                                         │
│     ├─ 声音：（高亢/低微）                       │
│     └─ 气味：（无/腥/臭）                        │
│                                                 │
│  3️⃣ 问诊                                         │
│     ├─ 主诉：主要不适                            │
│     ├─ 现病史：发病经过、症状特点                │
│     ├─ 既往史：既往疾病                          │
│     ├─ 月经史：周期、经量、痛经                  │
│     ├─ 生育史：孕产次数、哺乳情况                │
│     ├─ 生活方式：作息、饮食、情绪                │
│     └─ 十问歌：寒热/汗/头身/便溺/饮食/胸腹/聋    │
│                                                 │
│  4️⃣ 切诊                                         │
│     └─ 脉诊：（弦/滑/涩/细/数/迟/虚/实）         │
│                                                 │
│  5️⃣ 辨证分析                                      │
│     ├─ 八纲辨证：阴阳/表里/寒热/虚实             │
│     ├─ 脏腑辨证：肝/脾/肾/冲任                   │
│     ├─ 气血津液辨证：气滞/血瘀/痰凝              │
│     └─ 综合判断：证型名称                        │
│                                                 │
│  6️⃣ 治则治法                                      │
│     └─ 疏肝理气 / 健脾化痰 / 活血化瘀...          │
│                                                 │
│  7️⃣ 方药                                          │
│     └─ 方剂 + 加减                               │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 2.2 当前系统 vs 标准流程对比

| 辨证步骤 | 标准要求 | 当前系统 | 差距评估 |
|---------|---------|---------|---------|
| 望诊 | 舌象、面色、形体 | ❌ 无 | 🔴 完全缺失 |
| 闻诊 | 声音、气味 | ❌ 无 | 🔴 完全缺失 |
| 问诊 | 十问歌、症状清单 | ⚠️ 部分（仅主诉） | 🟡 严重不足 |
| 切诊 | 脉象 | ❌ 无 | 🔴 完全缺失 |
| 辨证分析 | 八纲/脏腑/气血 | ❌ 无 | 🔴 完全缺失 |
| 治则治法 | 基于证型 | ⚠️ 简单映射 | 🟡 无推理 |
| 方药 | 辨证论治 | ⚠️ 查表 | 🟡 无个体化 |

**结论**：当前系统缺失中医辨证**70% 以上的核心信息**，无法进行可靠的中医诊断。

## 3. 误导风险评估

### 3.1 临床风险

| 风险等级 | 风险描述 | 发生概率 | 影响程度 |
|---------|---------|---------|---------|
| 🔴 高风险 | 无证型依据却显示中医诊断 | 高 | 严重 |
| 🔴 高风险 | 医生可能信赖无证型依据的诊断 | 中 | 严重 |
| 🟠 中风险 | 患者接受不准确的中医治疗 | 中 | 中等 |
| 🟠 中风险 | 延误正规诊疗 | 低 | 严重 |

### 3.2 具体误导场景

#### 场景 1：无证型依据

```
医生操作：
1. 创建患者：张三，35 岁
2. 填写基本信息时，随意选择"气郁质"（未进行体质评估）
3. 系统存储：constitution = "气郁质"

系统输出：
📋 中医诊断
- 体质：气郁质 ❌（无评估依据）
- 证型：肝郁气滞 ❌（无辨证过程）
- 方剂：逍遥散 ❌（无处方依据）

风险：
医生可能将此作为诊断依据，给患者开具逍遥散，但患者实际可能是"痰湿质"或"血瘀质"。
```

#### 场景 2：Copilot 虚构数据

```
医生询问："这个患者的中医证型是什么？"

Copilot 回复：
📋 四诊信息
- 症状：乳房胀痛，经前加重
- 舌象：舌红，苔薄黄 ❌（系统根本没有舌象数据！）
- 脉象：弦细 ❌（系统根本没有脉象数据！）

🔍 辨证分析
- 证型：肝郁气滞证

风险：
医生可能误以为系统真的采集了舌象脉象，相信辨证结果。
```

#### 场景 3：AI 推理假象

```
医生点击"AI 中医辨证"

系统处理：
1. 读取患者年龄：35 岁
2. 读取超声结果：BI-RADS 3 类
3. 查表映射：35 岁 + BI-RADS 3 → "肝郁气滞"（无医学依据）
4. 返回结果：证型 = "肝郁气滞", 置信度 = 85%（虚构）

风险：
给简单规则披上 AI 外衣，误导医生认为有科学依据。
```

## 6. 建议

### 强烈建议（优先级最高）

**立即移除所有中医相关字段和功能**，理由：

1. **无辨证依据** - 缺少舌象、脉象等核心信息
2. **误导医生** - 显示无证型依据的诊断结果
3. **法律风险** - 可能被视为提供虚假医疗信息
4. **专业声誉** - 降低系统可信度
5. **资源浪费** - 维护无用的代码和数据

### 可选方案

如果确实需要中医功能：
1. **明确标注**："中医功能需配合四诊信息采集使用"
2. **单独模块**：与当前超声 AI 系统完全独立
3. **专家参与**：聘请中医专家全程指导
4. **临床验证**：进行前瞻性临床研究

中医辨证是一个严谨的医学过程，需要完整的四诊信息采集和专业的辨证推理。仅凭超声影像和基本人口学信息，无法进行任何可靠的中医诊断。

如果未来确实需要中医功能，必须：
1. 补全四诊信息采集
2. 建立标准化评估量表
3. 聘请中医专家参与
4. 进行临床验证

**不要将简单规则包装成 AI，不要用下拉选择替代辨证过程，不要给医生提供无证型依据的诊断结果。**
