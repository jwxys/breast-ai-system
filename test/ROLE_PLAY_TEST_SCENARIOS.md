# 角色模拟测试场景

## 测试目标
从不同用户角色的角度检测系统功能，发现潜在问题并修复。

---

## 👨‍⚕️ 角色 1: 乳腺科实习生 (张医生)

### 角色信息
- **姓名**: 张明
- **职位**: 乳腺科实习医生
- **工作年限**: 1 年
- **技术水平**: 中等，熟悉基本流程
- **使用场景**: 日常门诊、初诊筛查、随访管理

### 典型工作流

#### 场景 1: 新患者初诊
1. 登录系统
2. 创建新患者档案
3. 录入患者基本信息 (姓名、年龄、联系方式)
4. 记录主诉和现病史
5. 开具超声检查
6. 上传超声图像
7. 等待 AI 辅助诊断结果
8. 查看 AI 分析报告
9. 生成初步诊断
10. 提交上级医生审核

**关注点**:
- 操作流程是否清晰
- 信息录入是否便捷
- AI 结果是否易懂
- 是否有引导提示

#### 场景 2: 患者随访
1. 查询患者列表
2. 筛选需要随访的患者
3. 创建随访记录
4. 录入随访检查结果
5. 对比历史数据
6. 更新诊断结论
7. 安排下次随访

**关注点**:
- 随访提醒是否明显
- 历史数据对比是否直观
- 随访计划是否灵活

#### 场景 3: 学习知识库
1. 浏览知识库分类
2. 搜索相关病例
3. 查看 AI 诊断依据
4. 学习 BI-RADS 分级标准

**关注点**:
- 知识库分类是否清晰
- 搜索是否准确
- 内容是否实用

### 角色信息

- **技术水平**: 熟练，注重效率
- **使用场景**: 会诊审核、复杂病例、科研数据

### 典型工作流

#### 场景 1: 审核下级医生诊断
1. 查看待审核列表
2. 审阅患者完整病历
3. 查看 AI 辅助诊断结果
4. 对比下级医生诊断
5. 修改/确认诊断
6. 签署审核意见
7. 发布报告

**关注点**:
- 待办事项是否醒目
- 关键信息是否一目了然
- 修改操作是否便捷
- 审核记录是否可追溯

#### 场景 2: 复杂病例会诊
1. 调取患者全部资料
2. 查看多次随访记录
3. 对比影像变化
4. 参考 AI 多模态诊断
5. 查看中医证型分析
6. 制定综合治疗方案
7. 记录会诊意见

**关注点**:
- 数据整合是否完整
- 多模态结果是否可靠
- 治疗方案是否个性化
- 会诊记录是否规范

#### 场景 3: 科研数据分析
1. 导出特定条件病例
2. 查看统计数据
3. 分析 AI 诊断准确率
4. 导出研究数据

**关注点**:
- 数据导出是否方便
- 统计维度是否丰富
- 数据格式是否规范
- 隐私保护是否到位

#### 场景 4: 查看 AI 模型性能
1. 查看模型权重来源
2. 查看性能指标
3. 查看推理记录
4. 评估 AI 可信度

**关注点**:
- 模型信息是否透明
- 性能数据是否可信
- 推理过程是否可解释

## 🔍 检测重点

### 功能完整性
- [ ] 所有菜单能否正常访问
- [ ] 所有表单能否正常提交
- [ ] 所有数据能否正常保存
- [ ] 所有API 能否正常调用

### 用户体验
- [ ] 操作流程是否顺畅
- [ ] 错误提示是否清晰
- [ ] 加载状态是否有提示
- [ ] 空数据是否有引导

### 数据准确性
- [ ] 表单验证是否完善
- [ ] 数据格式是否正确
- [ ] 计算逻辑是否准确
- [ ] 关联数据是否一致

### 边界情况
- [ ] 空数据处理
- [ ] 超长文本处理
- [ ] 特殊字符处理
- [ ] 网络异常处理

### 医疗规范
- [ ] BI-RADS 分级是否标准
- [ ] 医学术语是否规范
- [ ] 报告格式是否合规
- [ ] 审核流程是否严谨
