# 代码注释规范

本项目使用中文注释，确保代码可读性和可维护性。

## 📝 注释原则

1. **为什么 (Why)** - 解释代码的目的和原因
2. **做什么 (What)** - 说明代码的功能
3. **怎么做 (How)** - 复杂逻辑的实现方式
4. **注意事项** - 特殊情况和陷阱

## 📋 注释类型

### 1. 文件头注释

每个文件开头说明用途：

```python
"""
患者管理模块 - API 接口层

提供患者信息的 CRUD 操作接口
"""
```

```typescript
/**
 * 患者管理模块 - 主页面
 * 
 * 显示患者列表和提供操作入口
 */
```

### 2. 函数/方法注释

说明参数、返回值和异常：

```python
async def create_patient(name: str, age: int):
    """
    创建新患者记录
    
    Args:
        name: 患者姓名
        age: 年龄
    
    Returns:
        dict: 创建的患者信息
    
    Raises:
        HTTPException: 创建失败时抛出
    """
```

### 3. 行内注释

复杂逻辑旁边添加简短说明：

```typescript
// 年龄排序
sorter: (a, b) => a.age - b.age,

// 操作按钮
<Button icon={<EditOutlined />} />
```

### 4. 配置注释

配置文件要详细说明每项含义：

```bash
# ========== 数据库配置 ==========
DATABASE_URL=mysql://root:pass@localhost:3306/breast_ai
```

## ✅ 注释检查清单

- [ ] 文件头是否有说明
- [ ] 公共函数是否有文档字符串
- [ ] 复杂逻辑是否有解释
- [ ] 配置项是否有说明
- [ ] 注释是否为中文
- [ ] 注释是否简洁明了

## 🚫 避免的注释

### 不要重复代码

```python
# ❌ 错误示例
x = x + 1  # x 加 1

# ✅ 正确示例
x = x + 1  # 计数器递增，用于统计访问次数
```

### 不要注释废弃代码

```python
# ❌ 错误示例
# old_function()  # 旧函数已废弃
new_function()

# ✅ 正确示例
new_function()
```
